# 作って理解する OS を x ヶ月かけてやっと読み終えたので感想をかく

普段はクライアント寄りのレイヤで仕事をしていますが低レイヤの知識が乏しいことにプログラマになってからずっとコンプレックスがあります。Swift というプログラミング言語そのものに興味を持ってからは一層強くそれを感じるようになりました。何より面白そうなのに手を出していないのはもったいないと思っていたので、この本が世に出た時にすぐに購入していたのですがあまりの分厚さになかなか手を付けられていなかったです。

この会社に来てからはじっくりとインプットの時間を作ることができるようになったので本来の生業としている領域のインプットとは別に毎日少しずつこの本を読み進めることにしました。

大体 X ヶ月で読み終えたので概要と感想を書きます。

## 概要とメモ

### ハードウェアの基礎

N 進数から始まり CPU、メモリとハードウェアを理解するための基礎的な部分に関する章です。少しずつ読み進めていたので気になったワードがあれば個別でググって様々なサイトのコンテンツを読んだので時間がかかりました。例えば CPU 内部には演算を行うための ALU があります。ALU は複数の値を演算する時など同時に一つより多くの聞く領域が必要になる場合があります。そのための領域がレジスタがあります。CPU の IC チップに複数のコアがある CPU をマルチコア CPU と呼びますが、このコアは ALU とレジスタで構成される 1 組のセットをそう呼んでいます。マルチコア CPU の内部にある複数のコアは外部接続を行う信号は共有なので性能を最大限に引き出すにはソフトウェア側でサポートが必要になります。このように元々知っていたが解像度が低かった知識について掘り下げたりうろ覚えだったもの同士がつながっていく感覚が楽しかったです。
